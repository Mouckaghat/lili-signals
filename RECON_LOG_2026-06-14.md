## (a) Bugs fixed

### BUG-1 — Lineup `source` type rejected real data → typecheck broken
- **Where:** `lib/lineupData.ts` (lines ~875, ~1471) + the `MatchLineup` type.
- **Symptom:** `tsc` errors — `Type '"api-football"' is not assignable to type '"sofascore" | "fotmob" | "baseline"'`. Committed data already used `"api-football"` as a source, but the type union didn't allow it.
- **Root cause:** the source union was never widened when api-football data was introduced.
- **Fix:** added `'api-football'` to the `source` union in both the generator (`scripts/sync-lineups.ts`) and the generated file (`lib/lineupData.ts`). File regenerated, so the data and type now agree.

### BUG-2 — Duplicate object keys in the name map
- **Where:** `scripts/sync-lineups.ts` (old lines 80–81).
- **Symptom:** `tsc` errors TS1117 — "An object literal cannot have multiple properties with the same name." `'Korea Republic'` and `"Côte d'Ivoire"` were each declared twice in `NAME_MAP`.
- **Root cause:** leftover "Fotmob variants" block duplicated keys already present above.
- **Fix:** de-duplicated `NAME_MAP` to one clean set of mappings.

### BUG-3 — Lineups never loaded (blocked data source)
- **Where:** `scripts/sync-lineups.ts` (whole fetch path).
- **Symptom:** every run logged `HTTP 403` from SofaScore and 0 lineups from Fotmob → 0 real lineups, no coaches. The dataset had coaches for only 2 of 8 played matches.
- **Root cause:** the bot depended on SofaScore/Fotmob public APIs, which now reject CI/dev requests (403).
- **Fix:** rewrote the bot to pull lineups from **api-football** `/fixtures/lineups` (formation + startXI + substitutes + coach) — the feed we already use for results and events. Baseline fallback kept for fixtures with no posted lineup yet.
